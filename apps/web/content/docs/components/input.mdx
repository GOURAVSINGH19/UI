## Overview

`Input` is a lightweight wrapper around the native `<input>` that ships with the
same tokens and shadows used across the rest of the system. It keeps spacing,
typography, and focus rings consistent so you can drop it into auth screens,
filters, or command bars without restyling from scratch.

## When to use

- Email or password collection inside onboarding flows
- Inline filters, global search, or quick actions
- Small form rows where you need custom visuals beyond the browser default

<Callout type="info" title="Component Usage">
  The Input component is designed to be flexible and theme-aware, making it easy to integrate into any form or search interface.
</Callout>

## Accessibility

- Associate inputs with `<label>` elements for better screen-reader context.
- Keep contrast on text and placeholder above WCAG AA (we default to 60%+).
- Disabled inputs remove pointer events, so pair with server-side guards.
