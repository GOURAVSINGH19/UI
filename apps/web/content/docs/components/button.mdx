## Overview

`Button` wraps the base shadcn/ui primitive and ships with a subtle gradient,
micro-interactions, and theme-aware tokens so you can drop it into any page
without fiddling with Tailwind classes. Use it for form submits, CTAs, or any
action that needs more visual weight than plain text links.

## When to use

- Primary call to action on a page or card
- Secondary outline button when paired with a solid primary action
- Icon trigger for compact surfaces such as toolbars

All other props pass straight through to the native `button` element, so you can
set `type="submit"`, `disabled`, `aria-*` attributes, and event handlers as
needed.

## Accessibility

- Always pair icon-only buttons with `aria-label`
- Disabled buttons set `aria-disabled` and skip pointer events, so rely on the
  `onClick` guard server-side as well
- Focus ring follows the theme accent color and respects reduced motion
