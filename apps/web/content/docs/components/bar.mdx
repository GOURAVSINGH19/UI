## Overview

`NavbarLinks` is a lightweight, animated navigation bar featuring a smooth
spring-based sliding indicator that tracks the currently active link. It is
built for Next.js apps using `next/link` and powered by `motion` for friction-based
animations.

With zero external dependencies beyond Motion and Tailwind, `NavbarLinks`
offers a sleek and interactive header element ideal for dashboards,
landing pages, or multi-section layouts.

## Installation

Copy the component into your project directory .

## Anatomy

- **Container** — wraps all nav items and ensures the indicator positions correctly.
- **Indicator (`motion.div`)** — animated highlight that moves under the active item.
- **Nav Items** — each item can be a plain label or a `next/link` route with optional icon.
- **Refs Map** — stores DOM references to compute indicator width + position.

## Customizing

- Pass any number of `items` with `id`, `title`, `href`, and optional `icon` nodes.
- Change the default active item using `initialActive`.
- Override colors, spacing, or hover states using the `className` prop or Tailwind utilities.
- Replace Motion spring stiffness/damping to modify animation feel.
- Icons can be any React node (Lucide, Radix, custom SVG, etc.).

## Tips

- Ideal for headers with 3–7 items to keep indicator motion feeling snappy.
- When used on dynamic pages, pair `active` state with router path to auto-sync selection.
- Icons should use consistent width to avoid indicator jitter during layout shifts.
- You can wrap the entire navbar in a parent container with blurred or elevated backgrounds
  to match modern dashboard UI patterns.
- If you add horizontal scroll or responsive collapsing, ensure the indicator recalculates using layout effects.
